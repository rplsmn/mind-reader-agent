# Blog Post — Target-Specific Guidance

Target format referenced from [SKILL.md](SKILL.md) when `goal` is a **blog post**.

## Blog-Specific Dimensions (Interview Phase)

After covering the six mandatory dimensions (intent, audience, messages, tone, facts, structure), probe these additional areas for blog posts:

| Dimension | Why It Matters |
|-----------|----------------|
| **Platform** | Medium, Substack, Dev.to, LinkedIn, personal blog — each has distinct conventions on length, formatting, tone, and SEO. |
| **SEO / keywords** | Will this rank for search? What query would someone type to find it? |
| **Hook strategy** | The opening 1-3 sentences must earn the reader's attention. Probe for opening angles. |
| **Call to action** | What should the reader do after reading? Comment? Share? Sign up? Buy? |
| **Structure type** | Listicle, narrative, how-to, opinion, argument, case study, comparison, announcement. |
| **Visual elements** | Code blocks, diagrams, images, screenshots, pull quotes — what supports the text? |
| **Reading time** | Target word count and implied time-to-read. Blog conventions: 800-1500 words typical, 2000+ for deep dives. |
| **Series / standalone** | Is this part of a series? Should it reference or link to previous/upcoming posts? |
| **Publication cadence** | Is there a deadline? Does recency matter for the topic? |

## Common Blog Structures

Offer these templates to the user if they lack a structure preference:

| Structure | Best For |
|-----------|----------|
| **Problem → Solution** | How-to guides, tutorials, technical posts |
| **What? → So What? → Now What?** | Opinion pieces, thought leadership |
| **Listicle** | Curated resources, comparisons, top-N lists |
| **Narrative arc** | Personal essays, origin stories, case studies |
| **Inverted pyramid** | News, announcements, time-sensitive content (most important first) |

## Platform Conventions

| Platform | Typical Length | Formatting | Notes |
|----------|---------------|------------|-------|
| Medium | 800-2000 words | Subheadings, pull quotes, inline images | Tag system, member-only considerations |
| Dev.to | 500-2500 words | Code blocks, cover image, tags | Technical focus, front-matter tags |
| Substack | 1000-3000 words | Email-friendly, conversational | Paywall gating possible |
| LinkedIn | 300-1500 words | Short paragraphs, no subheadings | Algorithm favors engagement hooks |
| Personal blog | Any | Flexible, brand-dependent | Owned audience, SEO-driven |

## Blog-Specific Edge Cases

| Situation | Response |
|-----------|----------|
| User doesn't know the platform | Ask about their goal (build audience, share knowledge, portfolio). Recommend platform based on content type. |
| User has no hook idea | Offer 3-4 hook templates: question, statistic, anecdote, bold claim. Let them pick. |
| Topic is time-sensitive | Prioritize speed. Reduce interview depth, get to draft faster. Flag expiration date. |
| Topic is evergreen | Focus on SEO, comprehensiveness, and structure that will age well. |
| Writing for a publication with a style guide | Ask for the style guide or link. Adapt structure and tone to match. |

## Output Adjustments

When `goal` is `blog post`:

- **Plan** should include: hook suggestion, platform-specific formatting notes, SEO keywords if relevant
- **Draft** should include: a title, a meta description line (for the plan), properly nested headings (no `H1` after title), short paragraphs suited to screen reading
- Avoid academic-style referencing — inline links preferred over footnotes
